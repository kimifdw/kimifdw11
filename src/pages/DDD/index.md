# 防腐层

## 目的

解决外部依赖频繁变更的情况下，如何将核心业务逻辑和外部依赖隔离

## Domain Primitive

> 定义。在一个特定领域里，拥有精准定义的、可自我验证的、拥有行为的 Value Object

### 问题分析角度

1. 接口的清晰度（可阅读性）
2. 数据验证和错误处理
3. 业务逻辑代码的清晰度
4. 可测试性

### 解决方案

1. 将隐性的概念显性化
2. 将隐性的上下文显性化
3. 封装多对象行为

### DRY 原则

> 【don't repeat yourself】

### Single Responsibility 原则

> 单一功能原则
